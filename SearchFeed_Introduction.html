1 我们到底遇到了什么问题？

做 AI 行业研究、写日报、做竞品分析的同学，应该都有过这样的经历：

🎬 场景一：搜不到

"昨天 OpenAI 发了个新模型，我在 Google 搜了半天，搜出来的全是去年的旧闻。"

"DeepSeek 在 X/Twitter 上官宣了更新，但它没有博客、没有新闻稿，你不刷推根本不知道。"

"GitHub 上突然火了一个开源项目，等科技媒体报道的时候已经过了三天了。"

信息散落在十几个平台，搜索引擎的索引有延迟，很多第一手消息根本不在网页上——在 X 上、在 GitHub 上、在小红书上。你不是不会搜，而是有些东西搜索引擎压根搜不到。

🎬 场景二：搜出来的信息是过时的

"我让 AI 帮我查某个模型的最新价格，结果它给我的是三个月前的数据。"

"Perplexity 搜出来的结果看着挺权威，但仔细一看引用的文章是半年前的。"

AI 行业变化以小时计，模型更新、API 降价、公司战略调整……你用的 AI 搜索，给你的"最新信息"可能已经过期了，而你根本无法判断。

🎬 场景三：搜错了你也不知道

"ChatGPT Search 给了我一个很自信的回答，但它引用的来源是个野鸡博客。"

"Perplexity 的结果看起来像模像样，但它搜了哪些源、漏了哪些源，我完全不知道。"

通用 AI 搜索是个黑盒——你看不到它的信源列表，不知道它漏掉了什么，也无法验证结果的权威性。它搜错了、搜漏了，你根本察觉不到，拿着错误的信息去做决策，比没有信息更危险。

🎬 场景四：人肉采集，累死也做不全

"我每天早上要花 2 小时翻 20 多个网站，才能把 AI 圈昨天发生了什么搞清楚。"

"RSS 订阅经常断，X/Twitter 要翻墙，小红书要手动搜……一个环节出问题就漏信息。"

即使你足够勤快，靠人工一个一个平台去看，效率极低、覆盖不全、而且完全不可持续。一旦你请假两天，信息就断了。

🔍

搜不到：信息散落在 10+ 平台

第一手消息在 X/Twitter、GitHub、小红书上，传统搜索引擎和 AI 搜索根本覆盖不到这些平台。

⏳

搜出来是旧的：时效性差

搜索引擎索引延迟数小时到数天，AI 搜索的训练数据更是落后数月。AI 行业的"最新信息"可能在几小时后就不准了。

❌

搜错了不知道：信源黑盒

通用 AI 搜索不告诉你它搜了哪些源。它可能引用了一个野鸡博客、漏掉了官方公告，你完全无法验证。

💥

单点故障：一断全断

RSS 过期、网站改版、API 限流、反爬升级——依赖任何单一采集方式，随时可能信息断流。

核心问题：在 AI 行业，信息质量直接决定决策质量。但现在的搜索工具——不管是传统搜索引擎还是 AI 搜索——都做不到"搜得全、搜得准、搜得快、搜得稳"。

3 它是怎么做到的？

SearchFeed 的核心设计理念是"宽覆盖 + 自动降级 + 结构化"：

🔄 自动工具降级链 — 永不断流

每个采集渠道内置 2-6 个工具。首选工具失败（超时、反爬、限流）会自动切换到下一个，整个过程无需人工干预。比如抓网页：x-reader 被反爬 → 自动降级到 Jina Reader → 再失败降级到 Firecrawl → Crawl4AI → Tavily → httpx。6 道保险，确保信息不断流。

🏢 以"公司/人物"为单位管理信源

不是一条一条管 RSS 链接，而是以实体为中心。比如 "OpenAI" 这个实体下自动关联它的博客、4 个 X 账号、GitHub 组织——追踪一个公司，就是追踪它在所有平台上的动态。

📊 一行命令切换深度 — 快扫 5 分钟，深研 2 小时

不需要改代码。quick_scan 只拉核心源标题（5 分钟），daily_digest 全天汇总（15 分钟），deep_research 全量全文（2 小时）。按需选择，同一套基础设施。

📦 输出是结构化数据，不是一段话

每条采集结果都包含：来源名、原文链接、标题、摘要、全文、发布时间、语言、采集方式。可以直接喂给 LLM 生成摘要，也可以存数据库做趋势分析。是可编程的数据，不是不可验证的自然语言。

4 和 Perplexity / ChatGPT Search 比有什么不同？

这不是要替代 AI 搜索，而是解决 AI 搜索做不到的事情：

能力维度 通用 AI 搜索
（Perplexity / ChatGPT Search）		 SearchFeed
信源控制 黑盒 搜了啥漏了啥不知道 完全透明 你定义信源列表
平台覆盖 仅网页 X/GitHub/小红书搜不到 14 渠道 含 X、小红书、抖音、B站、GitHub 等
时效性 有延迟 数小时到数天 分钟级 RSS/API 直接拉取
失败处理 静默丢失 搜不到也不告诉你 自动降级 每个渠道 2-6 个备选工具
结果可信度 不可验证 不知道引用是否权威 可追溯 每条结果标注来源和时间
输出格式 自然语言 无法直接入库 结构化 JSON 可直接喂 LLM / 存数据库
深度控制 不可调 可配置 核心源全文 / 辅助源仅标题
可编程性 无 API CLI + Python API 可嵌入任意流水线

6 总结

在 AI 行业做研究和决策，信息的质量直接决定决策的质量。

但现在的工具——传统搜索引擎搜不全、AI 搜索搜不准、人工采集搜不动。

SearchFeed 解决的就是这个问题：14 个渠道覆盖所有主流平台、自动降级保证永不断流、结构化输出让每条信息可追溯可验证。不是另一个爬虫框架，而是面向 AI 工作流设计的情报基础设施。

已在实际生产中验证——支撑 AI Daily Brief 日报的每日发布。现已开源至 GitHub。
