核心理念：任务调度不按行业分类，而按信息密度分级。同一个行业（AI/手机/游戏/汽车），不同的问题需要完全不同的采集深度和信源范围。用户的一句话就能确定该走哪级密度。

信息密度分级详解

L1 · 点查（Point Query）— 一个事实，一个答案
⏱ 10秒~1分钟标题扫描1~3 源零成本▼

用户想确认一个具体事实：某款模型叫什么、某个功能上线了没、某条新闻是否真实。不需要全文抓取，不需要多信源交叉验证——快速找到答案即可。

典型案例

"GPT 最新的模型是什么？"
采集策略：读取 OpenAI 官方 Twitter/博客最新条目 → 返回模型名+发布日期
信源：OpenAI (T1P1) → 官方博客 RSS / @OpenAI Twitter
输出：一句话回答
"华为昇腾 910C 量产了吗？"
采集策略：Tavily Search → 取 Top 1 结果
信源：搜索引擎（无固定信源）
输出：是/否 + 来源链接
"三星 S26 什么时候发布的？"
采集策略：web_search 快速查询
输出：日期 + 来源

执行流程

① 识别目标实体→② 匹配信源或搜索→③ 读取最新条目标题→④ 返回事实答案

采集参数

  • 采集深度：title_only
  • 信源范围：目标实体的官方源 / 搜索 Top 1-3
  • 并发数：1~3
  • 最大条目：1~5
  • API 消耗：0~1 次搜索

输出形式

  • 一句话回答 + 来源 URL
  • 无需格式化报告
  • 直接在对话中返回
L2 · 快讯（Daily Scan）— 今天发生了什么
⏱ 2~10分钟标题+摘要10~50 源低成本▼

用户想了解某个领域/公司/行业今天的动态，需要扫描多个信源的最新标题，按相关性筛选，输出一份简短的列表。

典型案例

"AI 行业今天有什么新闻？"
采集策略：Tier 1-2 全部 RSS → feedparser 批量抓标题 → 按时间排序 → 取今日条目
信源：Tier 1 AI公司(30) + Tier 2 科技媒体(20+) + 聚合站(HN/Techmeme)
输出：10~30 条新闻标题列表，按类别分组
"华为今天有什么消息？"
采集策略：① 华为相关信源 RSS ② Tavily "华为 today" → 取今日结果
信源：36氪(T3) + 微信搜索 + 百度新闻 + Tavily
输出：5~15 条新闻
"游戏行业最近一周有什么大事？"
采集策略：游戏专题信源 RSS → 关键词过滤 → 按时间取最近7天
信源：GamesIndustry.biz, IGN, GameSpot, Game Developer
输出：按天分组的新闻列表

执行流程

① 确定领域/关键词→② 选信源集合→③ feedparser 批量抓标题→④ 时间+关键词过滤→⑤ 去重+排序→⑥ 输出列表

采集参数

  • 采集深度：title_only
  • 信源范围：领域相关 Tier 1-2 + 搜索补充
  • 并发数：15~20
  • 最大条目：50~200
  • API 消耗：0~3 次搜索

输出形式

  • 新闻列表（标题 + 一句话摘要 + 来源 + 时间）
  • 按类别或时间分组
  • JSON / Markdown / 对话内直接输出

对应代码 Profile: quick_scan / daily_digest

L3 · 专题（Topic Focus）— 围绕一个主题，全面了解
⏱ 10~30分钟标题+部分全文20~100 源中等成本▼

用户需要围绕一个特定主题/产品/公司全面收集信息，需要从多角度（官方/媒体/社区/分析）获取内容。核心信源要读全文，辅助信源标题即可。

典型案例

"帮我总结三星 S26 手机的全部信息"
采集策略：
  ① Tavily Search "Samsung Galaxy S26" → Top 10
  ② 科技媒体 RSS(The Verge/Ars Technica) → 关键词过滤
  ③ 核心评测文章 → x-reader 全文抓取
  ④ Reddit r/Android → 社区反馈
  ⑤ YouTube → yt-dlp 评测视频字幕
输出：结构化专题报告（规格/价格/评价/对比）
"AI Agent 最近有什么进展？整理一下"
采集策略：Tier 1 AI公司源关键词过滤 + Tavily Search + arXiv 论文 + 核心博客全文
输出：专题摘要（产品进展/学术突破/业界观点）
"帮我看看 Cursor 和 Windsurf 最近都更新了什么"
采集策略：官方 changelog/blog 全文 + Twitter 动态 + HN 讨论
输出：对比式更新日志

执行流程

① 解析主题关键词→② 匹配信源→③ RSS+搜索→④ 关键词过滤→⑤ 核心文章全文→⑥ 社区/视频补充→⑦ 结构化输出

采集参数

  • 采集深度：混合 — 核心源 full_text + 辅助源 title_only
  • 全文抓取：5~15 篇核心文章
  • 并发数：10
  • API 消耗：3~10 次搜索 + 5~15 次网页抓取

输出形式

  • 结构化专题报告（分章节）
  • 关键发现 + 详情 + 来源引用
  • HTML / Markdown

对应代码 Profile: topic_focus / signal_brief

L4 · 报告（Full Report）— 全景扫描 + 分析研判
⏱ 30分~2小时全文采集50~200 源较高成本▼

用户需要一份完整的调研报告，覆盖某个行业/领域的全貌。需要广泛采集、深度阅读、交叉验证、分析研判。

典型案例

"做一份关于海外休闲游戏的调研报告"
采集策略：
  ① 游戏专题信源 RSS 全扫 → 关键词过滤
  ② Tavily Research "海外休闲游戏市场 2026" → 深度搜索
  ③ 核心报道 20~30 篇全文抓取
  ④ Exa AI → 行业报告/数据
  ⑤ Reddit r/gamedev → 开发者视角
  ⑥ Product Hunt → 新游戏产品
  ⑦ YouTube → 行业分析视频字幕
输出：完整调研报告（市场概况/竞品/趋势/机会/风险）
"帮我做一份 AI 编程工具赛道分析"
采集策略：各工具官方源全文 + 融资报道 + Reddit/HN 用户评价 + GitHub Star 趋势 + 招聘岗位
输出：赛道分析报告（玩家图谱/产品对比/融资/用户口碑/趋势判断）
"整理一份 2026 年 2 月 AI 行业月报"
采集策略：全 Tier 全源全文模式，30天内所有内容
输出：Signal Brief 月报

执行流程

① 拆解研究框架→② 多维信源选择→③ RSS 全量扫描→④ 搜索补充→⑤ 全文抓取(20-30篇)→⑥ 社区/视频/招聘→⑦ 交叉验证→⑧ 报告输出

采集参数

  • 采集深度：full_text（全部核心文章）
  • 全文抓取：20~50 篇
  • 并发数：8（避免限流）
  • API 消耗：10~30 搜索 + 20~50 网页抓取

输出形式

  • 完整调研报告（5000~20000 字）
  • 分章节：概述/分析/数据/趋势/建议
  • HTML 报告（带图表、引用）

对应代码 Profile: deep_research / gaming_focus

L5 · 深研（Deep Research）— 长周期持续追踪 + 反复迭代
⏱ 数小时~持续全文+交叉验证全部信源高成本▼

最高密度。不是一次性任务，而是持续追踪、反复补充、多轮迭代。涉及竞品情报、投资决策、战略规划等需要极高信息完整度的场景。

典型案例

"持续追踪 OpenAI/Anthropic/Google 的模型发布节奏，做竞品情报"
采集策略：三家公司全部官方源持续监控 + 每日自动扫描 + 每周搜索补充 + 行业领袖观点 + 学术论文 + 付费内容(需BrowserWing)
输出：持续更新的竞品情报看板
"帮我建一个 AI 芯片赛道的信息追踪系统"
采集策略：定义关键词 → 全源监控 → 每日自动采集+去重+归档
输出：每日更新数据库 + 周报/月报

采集参数

  • 采集深度：full_text + 登录态内容
  • 执行方式：定时任务（cron）+ 增量采集
  • 并发数：8
  • 去重：全量历史去重（数据库）
  • API 消耗：持续消耗，需关注月度额度

关键风险

  • 🔴 API 额度可能不够持续使用
  • 🔴 付费内容需 BrowserWing + Cookie
  • 🟡 数据库需定期清理优化
  • 🟡 需定期检查信源有效性

如何判断用户需要哪级密度？

用户话语特征信号词→ 密度级别案例
问一个具体事实"是什么""什么时候""多少钱""有没有"L1 点查"GPT-5 发布了吗？""Llama 4 多大参数？"
了解今日/近期动态"今天""最近""有什么新闻""最新动态"L2 快讯"AI 行业今天有啥？""华为最近有什么消息？"
围绕主题/产品总结"总结""整理""盘点""有什么进展""对比"L3 专题"总结三星S26全部信息""对比 Cursor 和 Copilot"
要求完整调研/报告"调研报告""分析""行业研究""全景""市场"L4 报告"做一份海外休闲游戏调研报告""AI编程工具赛道分析"
长期追踪/竞品监控"持续追踪""监控""每天/每周更新""情报""看板"L5 深研"持续追踪三大模型厂竞品""建立AI芯片追踪系统"

* 当无法确定密度级别时，默认走 L3 专题——比 L2 深一层，但不会像 L4 那样消耗过多资源。
